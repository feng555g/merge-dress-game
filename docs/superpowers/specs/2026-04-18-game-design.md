# 合成+养成手机游戏 设计文档

**日期：** 2026-04-18  
**状态：** 已确认  
**引擎：** Godot 4 + GDScript  
**平台：** iOS / Android  

---

## 1. 游戏概述

一款以女主成长故事为驱动的合成养成手机游戏。玩家通过棋盘合并和配方合成获取材料，用材料给女主换装、装修她的房间，推进故事章节，逐步揭开女主的身世与过去。

**核心体验定位：** 休闲 + 中度，碎片时间可玩，也有足够深度支撑长期留存。

---

## 2. 核心玩法循环

### 主循环（每局 3-5 分钟）

```
进入关卡棋盘
  → 拖拽合并相同物品 → 生成高级物品
  → 收集指定目标物品（关卡任务）
  → 完成任务 → 获得星级评分（1-3星）+ 掉落材料
  → 材料进入背包
```

### 配方合成（关卡外）

```
背包材料 → 查看配方表 → 满足材料条件 → 触发合成
  → 产出服装单品 或 家具道具
  → 装备到女主（装扮系统）或 放置到房间（装修系统）
  → 满足章节解锁条件 → 触发剧情对话 → 解锁下一章
```

### 双合成系统说明

| 系统 | 触发位置 | 作用 |
|------|---------|------|
| 棋盘合并 | 关卡内 | 赚取原料材料 |
| 配方合成 | 关卡外（工坊界面） | 将材料制作成装备/家具 |

两者形成完整闭环：打关卡→得材料→合成道具→完成养成→解锁故事→解锁新关卡。

## 3. 故事与章节结构

### 世界观

女主是一个身世成谜的女孩，游戏开始时她独居在一间破旧的小屋里，衣着朴素。随着关卡推进，她与周围人物的关系、以及她真实的过去逐渐浮现。每章结尾留下悬念，驱动玩家继续。

### 章节模板

```
第 N 章
  ├─ 解锁条件：完成上一章养成目标
  ├─ 关卡组：5-8 个棋盘关卡
  ├─ 材料产出：该章对应的材料类型
  ├─ 养成目标：
  │    装扮 → 合成并装备指定服装（1-2件）
  │    装修 → 合成并放置指定家具（1-2件）
  ├─ 剧情触发：养成目标全部完成后，播放故事对话
  └─ 解锁下一章
```

### 前三章示例

| 章节 | 主题 | 装扮目标 | 装修目标 |
|------|------|---------|---------|
| 第1章 | 新的开始 | 日常裙装 | 旧木桌、布艺椅 |
| 第2章 | 神秘来信 | 出行外套 + 皮靴 | 书架、台灯 |
| 第3章 | 隐藏的秘密 | 晚礼服 | 梳妆台、壁画 |

## 4. 养成系统

### 装扮系统

- 女主模型展示在专属界面，支持实时预览换装效果
- 装扮槽位：上衣 / 下装 / 鞋子 / 配饰 / 发型（共 5 槽）
- 每章解锁 1-2 件新服装，通过配方合成获得
- 已获得的服装可以自由搭配切换

### 装修系统

- 房间分为：客厅 / 卧室 / 厨房 / 庭院（共 4 间）
- 每章解锁部分家具/装饰位置
- 家具放置后可拖动调整位置（自由摆放）
- 房间视角为 2.5D 等距视角（类似 Homescapes 风格）

### 材料分类

| 材料类型 | 主要用途 | 主要来源关卡 |
|---------|---------|------------|
| 布料/丝线 | 服装合成 | 早期章节 |
| 宝石/饰品零件 | 配饰合成 | 中期章节 |
| 木材/石料 | 家具合成 | 贯穿全程 |
| 稀有材料 | 高级道具合成 | 高星评分奖励 |

## 5. 技术架构

### 引擎与语言

- **引擎：** Godot 4.x
- **语言：** GDScript
- **UI：** Godot Control 节点系统
- **数据：** JSON 配置文件 + Godot Resource（.tres）

### 模块划分

```
res://
  ├─ scenes/
  │    ├─ MergeBoard/      # 棋盘合并场景
  │    ├─ Workshop/        # 配方合成界面
  │    ├─ DressUp/         # 装扮界面
  │    ├─ Room/            # 房间装修界面
  │    ├─ Story/           # 剧情对话场景
  │    └─ MainMenu/        # 主菜单/章节选择
  ├─ scripts/
  │    ├─ MergeBoard.gd    # 棋盘逻辑
  │    ├─ CraftSystem.gd   # 配方合成逻辑
  │    ├─ StoryManager.gd  # 章节/剧情管理
  │    ├─ DressUpSystem.gd # 装扮系统
  │    ├─ RoomSystem.gd    # 装修系统
  │    ├─ Inventory.gd     # 背包/材料管理
  │    └─ SaveSystem.gd    # 本地存档
  ├─ data/
  │    ├─ levels/          # 关卡配置 JSON
  │    ├─ recipes/         # 配方表 JSON
  │    ├─ items/           # 物品定义 JSON
  │    └─ story/           # 剧情对话 JSON
  └─ assets/
       ├─ characters/      # 女主立绘/装扮贴图
       ├─ items/           # 棋盘物品图标
       ├─ furniture/       # 家具贴图
       └─ ui/              # 界面元素
```

### 数据驱动原则

关卡配置、配方表、故事对话全部外置为 JSON，新增内容不修改代码，只添加数据文件。

### 存档

本地存档使用 Godot 内置 `FileAccess` 写入 JSON，存储：章节进度、背包状态、装扮状态、房间状态。

## 6. 美术风格

- 参考消消乐（Candy Crush）风格：明亮饱和、圆润光泽、卡通质感
- 棋盘物品：圆润宝石/糖果类图标，有高光和阴影
- 女主角色：Q 版或半写实二头身，表情生动
- 房间：2.5D 等距视角，色彩温暖，细节丰富
- UI：圆角卡片风格，配色明快

---

## 7. 社交功能

- 轻度排行榜：本周合成数量 / 关卡星级总分
- 单机为主，排行榜不影响核心体验
- 暂不实现好友系统和互赠功能

## 8. 开发范围（MVP）

当前阶段目标：完成可玩的单机主体。

**包含：**
- 完整棋盘合并玩法（含关卡系统）
- 配方合成系统
- 装扮系统（换装预览）
- 装修系统（家具放置）
- 故事章节系统（对话、解锁）
- 本地存档
- 前 3 章完整内容

**不包含（后续迭代）：**
- 广告接入
- 内购系统
- 服务端 / 账号系统
- 完整章节内容（3章后续扩展）
